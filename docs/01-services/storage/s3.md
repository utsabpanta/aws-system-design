# S3 (Simple Storage Service)

> **One-line summary.** Internet-scale object storage. The default substrate for almost every cloud-native workload — data lakes, backups, static sites, logs, ML datasets, application assets.

## TL;DR

- The single most important AWS service. If a workload touches files larger than a row in a database, the right home for those bytes is usually S3.
- Eleven nines of durability per object (Standard, IA, Glacier classes), 99.99% availability SLA on Standard. Cross-Region replication, versioning, and Object Lock for stronger guarantees.
- **Many storage classes**, each tuned to a different access pattern. Lifecycle policies (or **S3 Intelligent-Tiering**) move data between them automatically.
- **S3 Express One Zone** (directory buckets) is the high-performance single-AZ tier: single-digit-ms latency, up to 2M GET TPS, ~10× faster than Standard for hot data — at higher per-GB cost.
- **S3 Tables** (managed Apache Iceberg tables) is a first-class AWS resource for tabular analytics workloads.
- The biggest operational risks are all about access: public buckets, leaked URLs, KMS key sprawl. Block Public Access on at the account level is non-negotiable.

## When to use it

- Data lake / analytics raw zones (with Athena, EMR, Glue, Redshift Spectrum, S3 Tables).
- Application assets, user-uploaded media, ML datasets, model artifacts.
- Logs, audit trails, CloudTrail, ELB / CloudFront / VPC Flow Logs.
- Backups (RDS, EBS, application-level), DR replicas.
- Static website hosting (paired with CloudFront).
- Anywhere you'd reach for an NFS / SMB share but don't actually need POSIX semantics — S3 + a small SDK wrapper is usually simpler and orders of magnitude cheaper.

## When NOT to use it

- Random in-place updates within an object. Objects are immutable — you replace, not edit.
- A POSIX filesystem (locking, hierarchical permissions, append). Use EFS, FSx, or — for small POSIX-on-S3 needs — Mountpoint for Amazon S3.
- Latency-critical (< 1 ms) lookups for small records. Use DynamoDB, ElastiCache, or MemoryDB.
- Transactional workloads. S3 is strongly read-after-write consistent (since 2020) but has no multi-object transactions.

## Key concepts

**Bucket** — the top-level container. Globally unique name. Region-pinned (data never leaves the Region unless you replicate it). Two bucket types now exist:

- **General purpose buckets** — the classic, multi-AZ, all storage classes except S3 Express One Zone.
- **Directory buckets** — single-AZ, only for S3 Express One Zone; hierarchical namespace, much higher TPS.

**Object** — bytes + key + metadata. Up to 5 TB per object; multipart upload above 5 GB. Key is opaque to S3 but conventions (`/`-delimited prefixes) make it usable as a folder structure.

**Storage classes:**

| Class | Use case | Notes |
|---|---|---|
| **Standard** | Hot, frequently accessed data | Default; 99.99% SLA, three+ AZs. |
| **Intelligent-Tiering** | Unknown / changing access patterns | Auto-moves objects between Frequent, Infrequent, Archive Instant, Archive, Deep Archive tiers based on observed access. |
| **Standard-IA** | Infrequently accessed but needs ms retrieval | Cheaper storage, retrieval fee, min 128 KB billing, min 30-day storage. |
| **One Zone-IA** | Reproducible / non-critical infrequent data | Single AZ; cheaper still. |
| **Glacier Instant Retrieval** | Archive with ms retrieval | See [glacier.md](glacier.md). |
| **Glacier Flexible Retrieval** | Archive with minutes-hours retrieval | See [glacier.md](glacier.md). |
| **Glacier Deep Archive** | Long-term archive, 12–48 h retrieval | Cheapest. See [glacier.md](glacier.md). |
| **Express One Zone** | High-perf, single-AZ, latency-critical hot data | Directory buckets only. |

**Lifecycle policies** — rules that transition objects between classes or expire them after N days. The right way to age data into cheaper tiers without writing code.

**Versioning** — keep prior versions when an object is overwritten or deleted. Pair with **MFA Delete** and **Object Lock** for tamper-resistant audit logs.

**Object Lock** — WORM (write-once read-many). Two modes: Governance (admins can override with permission) and Compliance (no one can delete until retention expires). Used for regulatory backups and immutable audit logs.

**Replication** — Cross-Region Replication (CRR) for DR, Same-Region Replication (SRR) for cross-account / aggregation, Replication Time Control (RTC) for 15-minute RPO SLA, batch replication for backfills.

**S3 Tables** — first-class AWS resource for managed Apache Iceberg tables. Automatic compaction, snapshot expiration, and unreferenced-file cleanup. Table-level access control via IAM. Integrates with Athena, Redshift, EMR, Spark, and Glue catalog. Use it when you'd otherwise self-manage Iceberg tables in S3 — it's faster (≈3× query throughput, ≈10× TPS) and removes the maintenance toil.

**Mountpoint for Amazon S3** — open-source FUSE-style client that exposes S3 buckets as a POSIX-ish filesystem. Read-heavy, append-only workloads only — it does not implement full POSIX semantics. Now also available as an EKS CSI driver.

**Server-side encryption.** All new buckets get **SSE-S3** (AES-256) encryption by default automatically. **SSE-KMS** uses your KMS keys (recommended for sensitive data). **DSSE-KMS** dual-layer for FedRAMP / DoD workloads. Client-side encryption when AWS must not see plaintext.

**Block Public Access (BPA).** Four account-level toggles that block public bucket policies / ACLs. **Leave them all on**; the rare case where you need a public bucket is for CloudFront origin, which doesn't require public access (use Origin Access Control).

## Pricing model

- **Storage** — per GB-month, class-dependent. Standard is the most expensive; Glacier Deep Archive is the cheapest (often 100× cheaper).
- **Requests** — per 1,000 (PUTs more expensive than GETs). Glacier classes charge per-retrieval too.
- **Data transfer** — *into* S3 is free. *Out* to the internet is the big bill driver; *between AWS services in the same Region* is free for Gateway endpoint usage; cross-Region replication is per GB.
- **Lifecycle transitions** — per 1,000 objects.
- **Storage class minimums** — IA/Glacier classes have minimum-billable-size (128 KB) and minimum-storage-duration (30 / 90 / 180 days). Lifecycle-aging tiny short-lived objects can cost *more* than leaving them in Standard.
- **Intelligent-Tiering** has a small per-object monitoring fee; only worth it for objects > 128 KB.

## Quotas & limits

- **Buckets per account** — 100 default; up to 1,000 with a request.
- **Object size** — 5 TB max; multipart upload required > 5 GB.
- **Request rate per prefix** — 3,500 PUT/COPY/POST/DELETE per second, 5,500 GET/HEAD per second. Scales automatically by prefix; for very hot prefixes spread keys across them (random prefix, hash prefix, time-bucketed prefix).
- **Directory buckets (Express One Zone)** — 2M GET TPS and 200K PUT TPS per directory bucket; better than general-purpose for hot data.
- **Replication** — limited by source bucket request rate; RTC SLA is for 15-minute replication of 99.99% of new objects.

## Common pitfalls

- **Public buckets by accident.** Block Public Access at the account level prevents most of these. If you must serve from S3, use CloudFront + Origin Access Control instead of a public bucket.
- **Pre-signed URLs that outlive their purpose.** Default expirations and over-broad scope leak. Use short TTLs and `s3:prefix` conditions.
- **One bucket per environment per micro-service.** That's how an account hits the 100-bucket default. Use prefixes for separation; one bucket per major data domain.
- **No lifecycle policy on log buckets.** ELB / CloudFront / VPC Flow Logs grow forever otherwise. Transition to IA after 30 d, Glacier after 90, delete after retention.
- **Using Standard for everything.** Intelligent-Tiering is the right default for unknown access patterns; you'll pay a tiny monitoring fee in exchange for automatic cost optimization.
- **KMS key sprawl.** A new CMK per bucket per micro-service explodes complexity and cost (per-request KMS calls). One CMK per data classification per Region is usually right.
- **Ignoring S3 Express One Zone for hot workloads.** When the workload is per-object latency-critical (Spark shuffle, ML feature serving), Express One Zone is dramatically faster — and after 2025's price cuts, often cheaper per request.
- **Self-managing Iceberg tables in S3.** S3 Tables ships compaction, snapshot expiration, and cleanup for free; rolling your own usually loses on both performance and operations.
- **Multipart upload artifacts orphaned.** Failed multipart uploads leave parts that you keep paying for. S3 Storage Lens surfaces them; lifecycle rule with `AbortIncompleteMultipartUpload` cleans them automatically.

## Pairs well with

- [Glacier tiers](glacier.md) — the archive end of the S3 storage class spectrum.
- **CloudFront** — CDN in front for cheap, low-latency public delivery.
- **Athena / EMR / Glue / Redshift Spectrum / S3 Tables** — analytics directly on S3.
- **Lambda / EventBridge** — react to object creation/deletion events.
- **AWS Backup** — managed backups landing in S3.
- **Macie** — find PII in S3 buckets.
- **KMS** — encryption keys.
